## 四、总结与建议

### 已对齐

- **品牌与 VI**：点亮微光、色值、无「心光」残留。  
- **启动页**：三屏文案与按钮与设计方案/技术方案一致。  
- **危机熔断**：关键词、仅存时间戳/计数、全屏红、无拨号、勾选解除。  
- **退出逻辑**：熔断解除 + 双返回（Android Toast）。  
- **存储**：`@sparklight_`、KEYS 集中、不存 answers/危机细节。  
- **首次启动**：lastStage null →「去检测」；有值 → 仪表盘 + 今日卡片。  
- **24h**：危机冷却 + 检测防抖提示。  
- **概念查词**：GlossaryTerm + glossary.json。  
- **日期/清理/校验**：本地日期、90 天/50 条清理、contentValidation。

### 已修正

- **Android 包名**：`android/app/build.gradle` 中 **applicationId** 已改为 `com.lightspark.ignite`（与技术方案 §10.2 一致）；**namespace** 仍为 `com.helloworld`，以免 Java 包路径与 R 类引用变动引发构建问题。

### 建议补全（按优先级，可选）

1. **复学地图 / 百宝囊 / 家长港湾**：当前为占位内容，后续可接 roadmap 数据、解锁逻辑、假性复学预警、话术复制等。  
2. **夜间主题**：ThemeContext + useThemeSchedule 已实现，可接入 themeDay/themeNight 样式。  
3. **Zustand store**：可选，用于统一状态管理。  
4. **Lottie / Reanimated**：FlowerCelebration 当前为 Animated 降级版，可接入 Lottie 开花动画。
