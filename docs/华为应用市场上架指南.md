# 点亮微光 · 华为应用市场（AppGallery）上架指南

> 包名：`com.lightspark.ignite` · 应用名：点亮微光

---

## 一、上架前准备清单

### 1. 开发者账号

- 注册 [华为开发者联盟](https://developer.huawei.com/consumer/cn/)
- 完成**实名认证**（个人/企业）
- 同意应用市场相关协议

### 2. 应用包与签名

| 项 | 说明 |
|----|------|
| **APK** | 使用正式签名打 Release 包（见下文「生成上架用 APK」） |
| **版本** | `versionCode` 与 `versionName` 在 `android/app/build.gradle` 的 `defaultConfig` 中配置，每次上架新版本需递增 `versionCode` |

### 3. 隐私与合规

- **隐私政策链接**：应用市场必填。需准备一份可公网访问的隐私政策页面（说明数据收集范围、存储方式、不包含题目/答案等敏感内容落盘等），可放在官网或 GitHub Pages。
- 本应用为本地优先、评估结果与危机仅时间戳/计数，不存储题目与用户输入原文，可在隐私政策中明确说明。

### 4. 应用市场素材

- **应用名称**：点亮微光（与 `app.json` / `strings.xml` 一致）
- **应用简介**：一句话介绍（如：厌学阶段自检与复学路径指引，家庭教育辅助工具）
- **应用介绍/详情**：可包含功能说明、适用人群、免责声明
- **应用截图**：至少 2～5 张，建议包含：首页（心灯导引）、检测入口、点亮阶梯、呼吸练习等
- **应用分类**：根据实际选择，如「教育」或「健康」等
- **分发地区**：勾选需要上架的国家/地区（如仅中国大陆）

## 二、生成上架用 APK

### 方式 A：当前使用 Debug 签名（仅适合内测/体验）

当前 `android/app/build.gradle` 中 release 使用 debug 签名。可直接打 Release 包用于体验或部分渠道：

```powershell
cd 厌学检测-app
.\build-android.ps1
# 或
cd android
.\gradlew.bat assembleRelease
```

输出：`android/app/build/outputs/apk/release/app-release.apk`

### 方式 B：正式上架（推荐：使用 Release 签名）

1. **生成签名密钥（仅首次）**：

   ```bash
   keytool -genkeypair -v -storetype PKCS12 -keystore my-release-key.keystore -alias my-key-alias -keyalg RSA -keysize 2048 -validity 10000
   ```

   按提示设置密钥库密码、姓名、单位等。将生成的 `.keystore` 文件妥善备份并保密。

2. **在项目中配置 Release 签名**：

   在 `android/app/build.gradle` 的 `android { }` 中增加或修改：

   ```gradle
   signingConfigs {
       debug { ... }
       release {
           storeFile file('my-release-key.keystore')   // 或使用环境变量路径
           storePassword '你的密钥库密码'
           keyAlias 'my-key-alias'
           keyPassword '你的密钥密码'
       }
   }
   buildTypes {
       release {
           signingConfig signingConfigs.release  // 改为 release
           ...
       }
   }
   ```

   **安全建议**：不要将密码提交到 Git。可使用 `storeFile file(System.getenv("KEYSTORE_PATH") ?: "my-release-key.keystore")` 及环境变量传入密码，或使用 Android 的 keystore 环境变量方式。

3. **再次执行**：

   ```powershell
   cd android
   .\gradlew.bat assembleRelease
   ```

   使用生成的 `app-release.apk` 上传应用市场。

## 三、华为应用市场提交流程

### 1. 进入应用服务

- 登录 [华为开发者联盟](https://developer.huawei.com/consumer/cn/)
- 进入 **我的项目** → **应用** → [应用服务控制台](https://developer.huawei.com/consumer/cn/console#/serviceCards/AppService)

### 2. 创建并发布应用

| 步骤 | 操作 |
|------|------|
| 1 | 点击「添加应用」或「发布新应用」 |
| 2 | 选择「应用」（非游戏），填写应用名称「点亮微光」、包名 `com.lightspark.ignite` 等 |
| 3 | **上传 APK**：上传上一步生成的 `app-release.apk` |
| 4 | **基础信息**：语言、应用介绍、截图、分类等 |
| 5 | **分发信息**：付费模式（选免费）、分发国家/地区、**隐私政策链接**（必填） |
| 6 | 检查产品包、基础信息、分发信息无误后，**提交审核** |

### 3. 审核与反馈

- 审核通常 1～3 个工作日
- 若未通过：登录开发者联盟查看原因，按反馈修改后重新提交
- **咨询与申诉**：
  - 工单/反馈：[https://developer.huawei.com/consumer/cn/support/feedback/#/](https://developer.huawei.com/consumer/cn/support/feedback/#/)
  - 智能客服：[https://developer.huawei.com/consumer/cn/customerService/#/bot-dev-top/faq-top/faq-talk-top](https://developer.huawei.com/consumer/cn/customerService/#/bot-dev-top/faq-top/faq-talk-top)
  - 开发者论坛：[https://developer.huawei.com/consumer/cn/forum/](https://developer.huawei.com/consumer/cn/forum/)

## 四、常见问题

| 问题 | 建议 |
|------|------|
| 审核不通过：缺少隐私政策 | 准备可公网访问的隐私政策页面，在应用内「关于/设置」中也可提供链接 |
| 审核不通过：应用描述/分类不符 | 按实际功能填写介绍与分类，避免夸大或与教育/健康类规范冲突 |
| 包名已存在 | 本应用包名为 `com.lightspark.ignite`，若已在该账号下使用则无需更改；若冲突需换包名并重新配置工程 |
| 版本更新 | 每次上传新 APK 时，在 `build.gradle` 中提高 `versionCode`（如 2、3…）和 `versionName`（如 "1.1"） |
