## 二、核心视觉规范重申 (Visual Standards)

所有交付物必须严格遵守以下「光之宇宙」规范，违者需返工：

### 色彩铁律

- **背景**：严禁使用纯黑 (#000000) 或纯白 (#FFFFFF)。必须使用深空蓝 **#0F172A** 作为主基调。
- **光色**：数据表现必须使用渐变光色（Pale Ember **#FDE68A** → Warm Flame **#FB923C** → Pure Gold **#FACC15**）。
- **禁忌**：禁止出现刺眼的高饱和红/绿/蓝原色。

### 质感要求

- **柔和度**：所有边缘必须柔和，大量使用 High Gaussian Blur (高斯模糊) 营造光晕感。
- **通透感**：卡片容器必须体现 Glassmorphism (毛玻璃) 效果，禁止实心色块堆砌。
- **动态感**：静态图中需通过光影流向暗示「呼吸」和「流动」，避免死板。

### 性能约束

- Lottie 文件大小建议控制在 **200KB** 以内（复杂粒子可拆分或使用位图序列）。
- 切图资源需经过 Tinypng 等工具压缩，但在**保留透明度通道**的前提下。

## 三、验收流程 (Acceptance Process)

1. **初审 (Design Review)**：检查 Figma/Sketch 源文件的图层规范性、组件化程度；核对色值、字体、间距是否符合规范文档。
2. **动效预览 (Motion Preview)**：在真机上预览 Lottie 动画，检查流畅度、发热情况及是否掉帧；确认暗色模式下光效是否达到「温暖、希望」的心理预期。
3. **资源完整性检查**：对照本清单逐项核对文件数量、命名、格式。
4. **最终签字 (Sign-off)**：产品经理 + 技术负责人共同确认后，归档至 SVN/Git/蓝湖。
